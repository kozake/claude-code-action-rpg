## 1. ゲームループとは

Web アプリでは「ユーザーがボタンを押したら処理する」というイベント駆動が基本ですが、
ゲームは **毎秒60回、全ての状態を更新し続ける** という点が大きく異なります。

```
┌──────────────────────────────────────┐
│          ゲームループ (60fps)          │
│                                      │
│  1. 入力を読む (キーボード/タッチ)     │
│  2. ゲーム世界を更新する              │
│     - プレイヤー移動                  │
│     - 敵AI行動                       │
│     - 当たり判定 → ダメージ処理       │
│     - アイテム・パーティクル更新       │
│  3. 画面を描画する                    │
│                                      │
│  ↓ 約16.7ms後にまた1から繰り返し      │
└──────────────────────────────────────┘
```

### このゲームでの実装

```typescript
// src/main.ts
app.ticker.add((delta: number) => {
  scene.update(delta);  // ← 毎フレーム呼ばれる
});
```

PixiJS の `Ticker` がブラウザの `requestAnimationFrame` をラップしており、
約60FPSで `update()` を呼び出します。

**delta（デルタタイム）** は前フレームからの経過時間を表す値です。
60FPSなら `delta ≈ 1.0` になります。これを速度に掛けることで、
フレームレートが変動しても動きが一定になります。

```typescript
// 例: プレイヤーの移動
const mx = input.moveX * speed * dt;  // dt = delta / 60（秒単位に変換）
```

## 2. 技術スタック

| 技術 | 役割 | 備考 |
|------|------|------|
| **PixiJS** | 2D描画エンジン | Canvas/WebGL を抽象化。`Graphics` で図形描画、`Container` で階層管理 |
| **TypeScript** | 言語 | 型安全性でバグを防止 |
| **Vite** | ビルドツール | 開発サーバー (HMR) + 本番ビルド |
| **Web Audio API** | BGM/SE | 音声ファイルが無い場合の合成フォールバック付き |
| **GitHub Pages** | ホスティング | `docs/` ディレクトリからデプロイ |

### PixiJS の基本概念

このゲームを理解するために最低限知っておくべき PixiJS の概念:

- **`Application`**: ゲーム全体の管理者。Canvas要素、描画ループ(Ticker)を持つ
- **`Container`**: HTML の `<div>` に相当。子要素をグループ化し、まとめて移動・回転できる
- **`Graphics`**: Canvas の `ctx.fillRect()` に相当。図形をプログラムで描画する
- **`Text`**: テキスト描画。HPバーの数値やメッセージに使用
- **`RenderTexture`**: オフスクリーン描画。マップのタイル群を1枚のテクスチャにまとめる（高速化）

## 3. ファイル構成と責務

```
src/
├── main.ts                    # エントリーポイント（PixiJS初期化、ゲームループ開始）
├── input.ts                   # 入力管理（キーボード＋仮想ジョイスティック統合）
├── audio.ts                   # BGM/SE管理（ファイル再生＋Web Audio合成フォールバック）
│
├── game/
│   ├── GameScene.ts           # ★ ゲームの中枢。全システムの生成・接続・毎フレーム更新
│   ├── DungeonConfig.ts       # フロア定義（テーマ、色、敵ウェーブ構成）
│   ├── Skills.ts              # スキル定義（効果、ステータス変更）
│   ├── Items.ts               # アイテムシステム（ドロップ、拾得、効果）
│   ├── Projectile.ts          # 弾丸シミュレーション（敵弾＋プレイヤー弾）
│   ├── Particles.ts           # パーティクル演出（ヒット、死亡、環境）
│   ├── ScreenEffects.ts       # 画面演出（ビネット、フロア遷移、レターボックス）
│   │
│   ├── entities/
│   │   ├── Player.ts          # プレイヤー（移動、攻撃、レベルアップ、描画）
│   │   ├── Enemy.ts           # 敵の基底クラス（AI、ノックバック、HP管理）
│   │   ├── RangedEnemy.ts     # 遠距離攻撃する敵
│   │   ├── Boss.ts            # ボス（3フェーズ、特殊攻撃パターン）
│   │   └── SpecialEnemies.ts  # 特殊敵4種（突撃、爆弾、盾、召喚）
│   │
│   └── maps/
│       └── WorldMap.ts        # マップ生成・タイル管理・当たり判定
│
└── ui/
    ├── HUD.ts                 # HP/XPバー、メッセージ、コンボ表示、ボスHP
    ├── VirtualJoystick.ts     # スマホ用仮想ジョイスティック
    ├── AttackButton.ts        # スマホ用攻撃・スキルボタン
    ├── SkillSelect.ts         # レベルアップ時のスキル選択UI
    └── Minimap.ts             # 右上のミニマップ
```

### ファイルサイズの目安

最大のファイルは `GameScene.ts`（約850行の `update()` メソッドを含む）です。
これはゲーム全体の「接着剤」として、各システムを繋ぐ役割を持っています。

## 4. レイヤー構造

PixiJS の `Container` による描画の重ね順:

```
GameScene.container
├── world（ワールド座標系 — カメラで動く）
│   ├── WorldMap.container        # タイルマップ（最背面）
│   ├── Player.container          # プレイヤーキャラクター
│   ├── Enemy[].container         # 敵キャラクター群
│   ├── Boss.container            # ボス
│   ├── ParticleSystem.container  # パーティクル演出
│   ├── ProjectileSystem.container # 弾丸
│   └── ItemSystem.container      # ドロップアイテム
│
└── ui（画面座標系 — カメラの影響を受けない）
    ├── HUD.container             # HPバー、メッセージ
    ├── Minimap.container         # ミニマップ
    ├── VirtualJoystick.container # ジョイスティック
    ├── AttackButton.container    # 攻撃/スキルボタン
    ├── SkillSelectUI.container   # スキル選択
    └── ScreenEffects.container   # 画面エフェクト（最前面）
```

**重要な設計判断**: `world` と `ui` を分離することで、カメラ移動時に
ワールド内のオブジェクトだけが動き、UIは画面上の固定位置に留まります。

```typescript
// カメラ追従の実装（GameScene.update内）
this.world.x = -this.cameraX + this.screenW / 2;
this.world.y = -this.cameraY + this.screenH / 2;
// ui は動かさない → 画面に固定
```

## 5. ゲームの状態遷移

```
                    ┌─────────┐
                    │  title  │ タイトル画面
                    └────┬────┘
                         │ タップ / キー入力
                         ▼
                    ┌─────────┐
              ┌────▶│ playing │◀───────────────────┐
              │     └────┬────┘                     │
              │          │                          │
              │          ├── 敵を倒してレベルアップ    │
              │          ▼                          │
              │   ┌──────────────┐                  │
              │   │ skill_select │ スキル選択        │
              │   └──────┬───────┘                  │
              │          │ スキル選択完了             │
              │          └──────────────────────────┘
              │
              │     全ウェーブクリア（ボスフロア以外）
              │          │
              │          ▼ 階段出現 → 階段に触れる
              │   ┌──────────────────┐
              │   │ floor_transition │ フロア遷移演出
              │   └──────┬───────────┘
              │          │ 次のフロアを生成
              └──────────┘

              ┌─────────┐
              │ playing │ でHP が 0 になった場合
              └────┬────┘
                   ▼
             ┌──────────┐
             │ gameover │ ゲームオーバー
             └──────────┘

              ┌─────────┐
              │ playing │ でボスを倒した場合
              └────┬────┘
                   ▼
              ┌─────┐
              │ win │ クリア画面
              └─────┘
```

状態は `GameScene` の `state` プロパティで管理されます:

```typescript
type GameState = 'title' | 'playing' | 'skill_select' | 'gameover' | 'win' | 'floor_transition';
```

## 6. メインループの処理フロー

`GameScene.update(delta)` は毎フレーム呼ばれ、以下の順序で処理します:

```
GameScene.update(delta)
│
├── 1. 入力更新
│   └── input.update()  ← キーボード/ジョイスティックの状態を統合
│
├── 2. プレイヤー更新
│   └── player.update(dt, input, map)
│       ├── ダッシュ処理（スキルボタン）
│       ├── 移動 + 壁衝突判定
│       ├── ノックバック物理
│       ├── 攻撃クールダウン管理
│       └── 描画更新（向き、アニメーション）
│
├── 3. 敵更新（全敵をループ）
│   └── enemy.update(dt, player, map, fire)
│       ├── AI移動（プレイヤーに向かう）
│       ├── 攻撃判定（近接 or 遠距離）
│       ├── ノックバック + スタン
│       └── 特殊行動（突撃、自爆、召喚など）
│
├── 4. 戦闘判定
│   ├── プレイヤー攻撃 vs 敵（円-矩形交差判定）
│   ├── スキル攻撃 vs 敵（火炎旋風、ダッシュストライク）
│   ├── 敵弾 vs プレイヤー
│   ├── プレイヤー弾 vs 敵
│   └── ダメージ処理 → パーティクル → ノックバック → XP付与
│
├── 5. システム更新
│   ├── items.update()       ← アイテム拾得判定
│   ├── projectiles.update() ← 弾丸移動
│   ├── particles.update()   ← パーティクルアニメーション
│   └── screenEffects.update() ← 画面エフェクト
│
├── 6. ゲーム進行判定
│   ├── レベルアップ → スキル選択UI表示
│   ├── ウェーブクリア → 次ウェーブ or 階段出現
│   ├── 階段到達 → フロア遷移
│   ├── ボス撃破 → 勝利演出
│   └── プレイヤー死亡 → ゲームオーバー
│
├── 7. カメラ更新
│   ├── プレイヤー追従
│   └── 画面揺れ（screen shake）適用
│
└── 8. UI更新
    ├── hud.update()     ← HP/XPバー、メッセージ
    └── minimap.update() ← ミニマップ上のドット
```

## 7. システム間の依存関係

各システムが何に依存しているかを示します。
矢印は「AがBを使う」を意味します。

```
                        ┌────────────────┐
                        │   GameScene    │
                        │  (中枢・接着剤)  │
                        └───────┬────────┘
                                │ 全システムを所有・更新
          ┌─────────┬───────┬───┼───┬────────┬──────────┐
          ▼         ▼       ▼   ▼   ▼        ▼          ▼
      ┌───────┐ ┌──────┐ ┌───┐ │ ┌──────┐ ┌──────┐ ┌───────┐
      │Player │ │Enemy │ │Map│ │ │Items │ │Proj. │ │Partic.│
      └───┬───┘ └──┬───┘ └─┬─┘ │ └──────┘ └──────┘ └───────┘
          │        │       │   │
          │        │       │   ▼
          │        ▼       │ ┌──────────┐
          │     Player     │ │  Audio   │
          │    (攻撃対象)   │ └──────────┘
          │        │       │
          └────────┴───────┘
           Map (壁衝突判定)
```

### 依存のポイント

- **GameScene** は全システムを知っている唯一のクラス。他のシステムは互いを直接参照しない
- **Enemy → Player**: 敵AIがプレイヤー位置を参照して追尾・攻撃
- **Player, Enemy → WorldMap**: 移動時の壁衝突判定
- **GameScene → Audio**: ゲーム状態変化時にBGM切り替え
- **スキルの適用**: `Skills.ts` の `apply()` が `SkillStats` を変更 → `Player.syncSkills()` で反映

この設計により、個別システムの変更が他に波及しにくくなっています。
例えば `Particles.ts` を変更しても、`Player.ts` や `Enemy.ts` に影響はありません。
